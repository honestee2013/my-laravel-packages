Livewire DataTable Component

This is a reusable DataTable component built with Livewire for Laravel applications. It allows for customizable table fields, actions, and validation rules, supporting features like file uploads, select inputs, and text fields.

Features

  • Dynamic Fields: Add and configure different field types (text, select, textarea, file).
  • Custom Validation: Define validation rules for each field.
  • File Uploads: Supports file and image uploads with custom validation.
  • Actions: Simple and advanced actions for each row, such as edit, show, delete, etc.
  • Hidden Fields: Control visibility of fields on the table, form, and detail views.

Usage

Basic Example

<livewire:data-table
    model="App\Models\User"
    :fieldDefinitions="[        'photo' => [            'field_type' => 'file',            'validation' => 'nullable|image|max:1024|mimes:jpg,png,jpeg',        ],
        'name' => ['field_type' => 'text', 'validation' => 'required|string|min:10'],
        'email' => ['field_type' => 'email', 'validation' => 'required|email'],
        'password' => 'password',
        'about_me' => 'textarea',
        'location' => [            'field_type' => 'select',            'options' => ['Kano' => 'Kano', 'Kaduna' => 'Kaduna', 'Abuja' => 'Abuja'],
            'multiSelect' => true,
            'validation' => 'required|min:2'
        ]
    ]"
    :simpleActions="['edit', 'delete']"
    :moreActions="[        'show' => [            'title' => 'View Details',            'icon' => 'fas fa-eye text-primary',            'route' => 'users.show',        ]
    ]"
    :hiddenFields="[        'onTable' => ['password', 'password_confirmation'],
        'onEditForm' => ['password_confirmation'],
    ]"
/>

Parameters

model (required)

The model to be used for data fetching and manipulation.

fieldDefinitions (required)

An array that defines fields, their types, options, and validation rules.

  • field_type: The type of field (text, email, password, file, select, textarea).
  • validation: Laravel validation rules for the field.
  • options (for select fields): Available options for select dropdowns.
  • multiSelect (for select fields): Allows multiple selections if set to true.
  • selected (for select fields): Preselect options.

Example:

'location' => [    'field_type' => 'select',    'options' => ['Kano' => 'Kano', 'Kaduna' => 'Kaduna', 'Abuja' => 'Abuja'],
    'selected' => ['Abuja'],
    'multiSelect' => true,
    'validation' => 'required|min:2'
]

simpleActions (optional)

Array of actions (e.g., show, edit, delete) that are commonly used.

moreActions (optional)

Custom actions that require additional customization like icons, routes, etc.

'moreActions' => [    'edit' => [        'title' => 'Edit User',        'icon' => 'fas fa-edit',        'route' => 'users.edit',    ],
    'show' => [        'title' => 'View User',        'icon' => 'fas fa-eye',        'route' => 'users.show',    ]
]

hiddenFields (optional)

An array to hide specific fields in different contexts like the table, form, or detail views.

'hiddenFields' => [    'onTable' => ['password', 'password_confirmation'],
    'onDetail' => ['password'],
    'onEditForm' => ['password_confirmation']
]

Example of Defining Fields

'photo' => [    'field_type' => 'file',    'validation' => 'nullable|image|max:1024|mimes:jpg,png,jpeg'],
'about_me' => [    'field_type' => 'textarea',    'validation' => 'required|string|max:500'],
'location' => [    'field_type' => 'select',    'options' => ['Kano' => 'Kano', 'Kaduna' => 'Kaduna'],
    'multiSelect' => true,
    'validation' => 'required'
]

Methods

Field Customization

You can fully customize the fields through the fieldDefinitions parameter. This allows you to specify validation rules, field types, and options for fields like select inputs.

Action Customization

Both simple and custom actions are supported. Custom actions can include routes, icons, and labels. Actions can trigger redirects or modal popups.

Validation

Validation is based on the Laravel validation rules you define in the fieldDefinitions.

'photo' => [    'field_type' => 'file',    'validation' => 'nullable|image|max:1024|mimes:jpg,png,jpeg']

Notes

  • Ensure that the fieldDefinitions match the fields in your model.
  • The hiddenFields array allows you to manage field visibility across different parts of the component.
